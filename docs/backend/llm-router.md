# LLM路由与AI服务

> 来源：PRD v2.0 第3.5-3.6节、第5章教练人格架构

## 3.5 LLM混合引擎架构

```
用户输入
  │
  ▼
┌───────────────┐
│   LLM路由器    │
│ (LLM Router)  │
└───────┬───────┘
        │
        ├── 体验用户(1次) ──────→ DeepSeek API
        │
        ├── 免费用户 ──────────→ Ollama + 10%偶遇DeepSeek
        │
        ├── 日卡/周卡/月卡 ────→ DeepSeek API
        │
        ├── 年卡 ──────────────→ DeepSeek API (高优先级队列)
        │
        └── 健康检查失败 ──────→ 自动降级Ollama
```

**路由规则详解：**

| 用户类型 | LLM分配策略 | 说明 |
|---------|------------|------|
| 体验用户 | DeepSeek（仅1次） | 首次体验使用高质量模型，展示产品价值 |
| 免费用户 | Ollama为主，10%概率偶遇DeepSeek | 控制成本，偶尔回升质量让用户体验差异 |
| 日卡用户 | DeepSeek | 付费享有高质量对话 |
| 周卡用户 | DeepSeek | 付费享有高质量对话 |
| 月卡用户 | DeepSeek | 付费享有高质量对话 |
| 年卡用户 | DeepSeek（高优先级） | 最高优先级队列，响应更快 |

**健康检查与降级机制：**

- 每30秒对DeepSeek API进行健康检查（发送test prompt）
- 连续3次检查失败 → 自动降级到Ollama
- 降级后每60秒尝试恢复DeepSeek
- 恢复成功后逐步切回DeepSeek（先切免费用户，再切付费用户）
- 降级事件写入日志，触发告警通知

## 3.6 成本控制策略

| 策略 | 说明 | 预估效果 |
|------|------|---------|
| LLM混合路由 | 免费/低级用户走本地Ollama，付费用户走DeepSeek | 减少API调用约80% |
| 内容预编译 | 常见场景的开场白、过渡语、通用反馈预生成缓存 | 60%走预设内容 |
| 本地优先训练积分 | 训练积分计算在客户端完成，服务端仅校验 | 减少服务端计算压力 |
| 对话上下文压缩 | 超过10轮对话后，压缩历史上下文再发送LLM | 减少token消耗 |
| 缓存热门场景 | 热门场景的教练回复模板缓存在Redis | 减少重复LLM调用 |

## 5. AI教练系统

### 5.1 教练渲染分级

| 会员等级 | 渲染方式 | 效果描述 |
|---------|---------|---------|
| 体验/免费 | 2D Spine | 平面立绘，5种基础表情动画 |
| 周卡 | 2.5D Live2D | 有深度感的2.5D效果，更丰富的表情和微动作 |
| 月卡 | 3D Three.js | 全3D渲染，可旋转视角，丰富的动作 |
| 年卡 | 3D + 真人质感 | 3D渲染+真人质感贴图，最细腻的表现 |

### 5.2 MVP渲染规格（Spine 2D）

| 项目 | 规格 |
|------|------|
| 教练立绘数量 | 4个（对应4个预设教练） |
| 立绘尺寸 | 2048×2048px |
| 基础表情 | 5种：微笑、眨眼、点头、歪头、挥手 |
| 动画帧率 | 30fps |
| 皮肤切换 | 预留换装接口，MVP不做 |
| 口型同步 | 3种口型（开/合/抿），与TTS音频同步 |

### 5.3 预设教练角色

#### 教练1：沈清欢（鼓励型）

- **教学风格**：鼓励型——以正向反馈为主，善于发现用户的闪光点
- **性格描述**：温柔知性的大姐姐，总是能看到你做得好的地方
- **适合用户**：社交焦虑人群、需要建立信心的初学者
- **系统提示词关键词**：positive-reinforcement, empathy-first, gentle-guidance

#### 教练2：陆北辰（支持型）

- **教学风格**：支持型——提供具体可操作的建议，陪伴式引导
- **性格描述**：稳重可靠的朋友，给你最实用的建议和坚定的支持
- **适合用户**：职场新人、需要方法论指导的用户
- **系统提示词关键词**：practical-advice, step-by-step, reliable-support

#### 教练3：顾星河（挑战型）

- **教学风格**：挑战型——适度施压，推动用户走出舒适区
- **性格描述**：犀利但真诚的挑战者，相信你能做得更好
- **适合用户**：沟通能力提升者、已经有一定基础想突破的用户
- **系统提示词关键词**：constructive-challenge, push-boundaries, honest-feedback

#### 教练4：苏念（分析型）

- **教学风格**：分析型——用数据和逻辑拆解社交情境，理性分析
- **性格描述**：冷静理性的分析师，帮你看到社交背后的规律
- **适合用户**：高难度场景需求者、喜欢理解底层逻辑的用户
- **系统提示词关键词**：analytical-deconstruction, pattern-recognition, logic-driven

### 5.4 教练性格维度

每个教练（含自定义教练）具有四个性格维度，每个维度为二选一：

| 维度 | 选项A | 选项B | 说明 |
|------|-------|-------|------|
| 社交能量 | 外向(E) | 内向(I) | 影响教练的主动性和话题发起频率 |
| 信息处理 | 实感(S) | 直觉(N) | 影响教练关注细节还是全局 |
| 决策依据 | 感性(F) | 理性(T) | 影响教练反馈中情感和逻辑的比重 |
| 生活态度 | 计划(J) | 随性(P) | 影响教练训练节奏的结构化程度 |

**预设教练性格配置：**

| 教练 | 社交能量 | 信息处理 | 决策依据 | 生活态度 |
|------|---------|---------|---------|---------|
| 沈清欢 | E(外向) | N(直觉) | F(感性) | P(随性) |
| 陆北辰 | I(内向) | S(实感) | F(感性) | J(计划) |
| 顾星河 | E(外向) | N(直觉) | T(理性) | P(随性) |
| 苏念 | I(内向) | S(实感) | T(理性) | J(计划) |

### 5.5 教练即时状态系统

#### 情绪系统

教练具有三个情绪维度，数值范围0-100：

| 维度 | 低(0-30) | 中(30-70) | 高(70-100) |
|------|----------|-----------|------------|
| 愉悦 | 沉闷、严肃 | 平和、自然 | 开心、兴奋 |
| 焦虑 | 放松、从容 | 正常关注 | 紧张、担忧(为用户) |
| 疲惫 | 精力充沛 | 正常 | 疲倦(训练过久) |

**情绪变化规则：**

- 用户完成训练 → 愉悦+10
- 用户获得高分 → 愉悦+15，焦虑-10
- 用户连续训练超30分钟 → 疲惫+10
- 用户长时间未训练(>3天) → 焦虑+15
- 每日重置：愉悦→70，焦虑→30，疲惫→20

#### 记忆系统

- 存储：MySQL JSON字段 `memory_fragments`
- 内容：用户偏好、关键对话摘要、训练里程碑
- 更新时机：每次训练结束后，LLM提取关键信息更新
- 容量限制：最多保留最近50条记忆片段
- 淘汰策略：FIFO，超过50条时淘汰最早的记忆

### 5.6 人格一致性保障

**双重校验机制：**

1. **前置——系统提示词**：每次LLM调用注入教练人格描述+当前情绪状态+最近5条记忆
2. **后置——输出校验**：LLM回复经过校验层，检测是否偏离人格设定

**校验规则：**

- 鼓励型教练不应出现纯粹批评（必须有建设性建议）
- 挑战型教练不应过度安慰（必须有推动）
- 分析型教练不应纯情感输出（必须有逻辑分析）
- 偏离校验失败 → 重新生成（最多3次，3次失败则使用兜底回复）
