# 产品需求文档（PRD）：「对练社交」—— AI对话社交训练平台

**版本：v2.1（PRD v2.0后新增与变更版）**
**文档日期：2026-07-16**
**文档性质：开发基准文档，所有功能实现以此为准**

## 1. 文档概述与产品定位

### 1.1 产品定位

「对练社交」是一款 **AI驱动的对话社交训练App**，通过AI教练实时互动，结合CBT（认知行为疗法）、依恋理论、NVC（非暴力沟通）等心理学框架，为用户提供**结构化的社交技能训练**与**可量化的沟通能力提升**。

**定位于"社交技能训练平台"，不是"虚拟伴侣"或"情感陪伴"产品。**

### 1.2 一句话价值主张

> **"先在AI教练这里练一遍，再去面对真实世界。"**

### 1.3 核心差异化（vs 传统AI聊天）

| 维度 | 传统AI聊天 | 「对练社交」 |
|------|-----------|---------|
| **训练目标** | 无明确目标，泛泛而谈 | 每次对话有明确的社交能力训练目标 |
| **心理学框架** | 无 | CBT、依恋理论、NVC等心理学框架驱动 |
| **训练结构** | 自由聊天，无结构 | 场景化训练，有开始-过程-评估闭环 |
| **反馈机制** | 简单回复 | 多维度即时反馈+训练后评估报告 |
| **成长闭环** | 无成长追踪 | 能力雷达图+进步曲线+里程碑体系 |
| **形象联动** | 无 | 与"悦己颜值社"联动，社交自信与形象提升 |
| **合规设计** | 无实名认证/年龄分层/防沉迷/心理响应 | 实名认证 + 年龄分层 + 防沉迷 + 心理健康三级响应 |

### 1.4 技术目标

| 指标           | MVP目标                | 正式版目标        |
|----------------|------------------------|-------------------|
| AI推理响应时间 | < 3秒                  | < 2秒             |
| 教练渲染       | 2D动态立绘(Spine)      | 3D渲染 ≥ 30fps    |
| 语音延迟       | -                      | < 500ms           |
| DAU            | 1万                    | 10万              |
| 并发训练       | 500                    | 5000              |

## 2. 用户画像

### 2.1 核心用户

- **年龄段**：18-35岁都市年轻人
- **性别分布**：女性占比65-70%
- **地域**：一二线城市为主
- **消费能力**：中等以上，愿意为自我提升付费

### 2.2 四类典型用户

#### 用户A：社交焦虑人群（22-30岁）

- **特征**：在社交场合感到紧张不安，回避社交互动，担心被评价
- **核心需求**：在安全环境中练习社交，逐步建立信心
- **典型场景**：不敢参加聚会、害怕与陌生人交谈、在公共场合紧张
- **使用动机**：低成本试错，无真实社交压力

#### 用户B：职场新人（22-28岁）

- **特征**：初入职场，缺乏职场沟通经验，不知如何与上级/同事沟通
- **核心需求**：掌握职场沟通技巧，提升职业形象
- **典型场景**：面试紧张、汇报磕巴、不懂得体表达
- **使用动机**：职业发展刚需，希望快速提升

#### 用户C：沟通能力提升者（25-35岁）

- **特征**：有社交基础但希望精进，追求高阶沟通技巧
- **核心需求**：系统化提升沟通能力，学习特定技巧
- **典型场景**：想学谈判技巧、希望提升共情能力、改善亲密关系沟通
- **使用动机**：自我成长需求，愿意投入时间系统学习

#### 用户D：高难度场景需求者（22-40岁）

- **特征**：面临特定高难度社交场景，需要针对性训练
- **核心需求**：针对特定场景的实战演练
- **典型场景**：加薪谈判、投诉维权、砍价
- **使用动机**：迫切的现实需求，目标明确

### 3.4 情绪识别服务

情绪识别是驱动"树洞/建议/混合"三种模式的核心能力，通过多层分析引擎实现对用户情绪状态的精准判断。

#### 3.4.1 三层分析引擎

**第一层：文本情感分析**

| 分析维度 | 技术方案 | 说明 |
|---------|---------|------|
| 情感极性 | LLM内嵌情感分析能力 | 识别正面/负面/中性情感 |
| 情绪强度 | 基于关键词频率计算 | 0-100分，数值越高情绪越强 |
| 情绪类型 | 预设情绪词典匹配 | 支持6种核心情绪：快乐、平静、紧张、焦虑、沮丧、愤怒 |
| 语言风格 | 句式结构分析 | 识别感叹句、疑问句、否定句等 |

**第二层：行为分析**

| 分析维度 | 技术方案 | 说明 |
|---------|---------|------|
| 输入速度 | 计时分析 | 打字速度反映紧张程度 |
| 输入长度 | 字符计数 | 简短输入可能表示情绪低落 |
| 回复间隔 | 时间统计 | 长时间停顿可能表示思考或犹豫 |
| 选择模式 | 选项统计 | 频繁选择安全选项可能表示缺乏自信 |

**第三层：语音情绪分析（V1.1阶段）**

| 分析维度 | 技术方案 | 说明 |
|---------|---------|------|
| 语速 | 实时语音分析 | 过快/过慢可能表示情绪波动 |
| 语调 | 音高变化检测 | 语调起伏反映情绪强度 |
| 音量 | 分贝监测 | 异常音量可能表示激动或低落 |
| 停顿 | 间隔分析 | 频繁停顿可能表示紧张 |

#### 3.4.2 融合决策规则

```
用户输入
  │
  ├── 文本情感分析 ──→ 情感得分(0-100)
  │
  ├── 行为分析 ──────→ 行为特征向量
  │
  └── 语音分析(V1.1) → 语音特征向量
        │
        ▼
┌─────────────────────┐
│   融合决策引擎       │
│  加权融合计算        │
└──────────┬──────────┘
           │
           ▼
┌─────────────────────┐
│   模式判定           │
│  树洞/建议/混合      │
└─────────────────────┘
```

**加权融合公式：**

```
综合情绪得分 = 文本情感权重(60%) × 情感得分 +
              行为分析权重(30%) × 行为特征值 +
              语音分析权重(10%) × 语音特征值(V1.1)
```

**模式判定规则：**

| 综合情绪得分 | 判定模式 | 说明 |
|-------------|---------|------|
| 0-30 | 树洞模式 | 用户情绪低落，需要情感支持 |
| 31-70 | 建议模式 | 用户情绪正常，需要实用建议 |
| 71-100 | 混合模式 | 用户情绪复杂，需要情感支持+实用建议 |

#### 3.4.3 成本优化方案

| 策略 | 说明 | 预估效果 |
|------|------|---------|
| 缓存高频情绪模式 | 相同输入模式的情绪判定结果缓存5分钟 | 减少30%重复计算 |
| 分级计算 | MVP阶段：普通用户仅文本分析，VIP用户启用文本+行为两层分析；V1.1阶段：VIP用户启用完整三层分析（含语音分析） | 降低50%计算成本 |
| 异步计算 | 行为分析和语音分析(V1.1)异步执行，先返回基础判定结果 | 响应时间<1秒 |
| 本地计算 | 文本情感分析基础规则在客户端计算，复杂分析在服务端 | 减少60%服务端请求 |

### 3.5 AI服务

| 组件 | 技术选型 | 说明 |
|------|---------|------|
| LLM(主力) | DeepSeek-Chat | 高质量对话生成，用于付费用户 |
| LLM(兜底) | Ollama + Qwen2.5:14B | 本地部署，用于免费用户和降级场景 |
| TTS | Microsoft Azure TTS | 多音色语音合成 |

### 3.6 LLM混合引擎架构

```
用户输入
  │
  ▼
┌───────────────┐
│   LLM路由器    │
│ (LLM Router)  │
└───────┬───────┘
        │
        ├── 体验用户(1次) ──────→ DeepSeek API
        │
        ├── 免费用户 ──────────→ Ollama + 10%偶遇DeepSeek
        │
        ├── 日卡/周卡/月卡 ────→ DeepSeek API
        │
        ├── 年卡 ──────────────→ DeepSeek API (高优先级队列)
        │
        └── 健康检查失败 ──────→ 自动降级Ollama
```

**路由规则详解：**

| 用户类型 | LLM分配策略 | 说明 |
|---------|------------|------|
| 体验用户 | DeepSeek（仅1次） | 首次体验使用高质量模型，展示产品价值 |
| 免费用户 | Ollama为主，10%概率偶遇DeepSeek | 控制成本，偶尔回升质量让用户体验差异 |
| 日卡用户 | DeepSeek | 付费享有高质量对话 |
| 周卡用户 | DeepSeek | 付费享有高质量对话 |
| 月卡用户 | DeepSeek | 付费享有高质量对话 |
| 年卡用户 | DeepSeek（高优先级） | 最高优先级队列，响应更快 |

**健康检查与降级机制：**

- 每30秒对DeepSeek API进行健康检查（发送test prompt）
- 连续3次检查失败 → 自动降级到Ollama
- 降级后每60秒尝试恢复DeepSeek
- 恢复成功后逐步切回DeepSeek（先切免费用户，再切付费用户）
- 降级事件写入日志，触发告警通知

### 3.7 成本控制策略

| 策略 | 说明 | 预估效果 |
|------|------|---------|
| LLM混合路由 | 免费/低级用户走本地Ollama，付费用户走DeepSeek | 减少API调用约80% |
| 内容预编译 | 常见场景的开场白、过渡语、通用反馈预生成缓存 | 60%走预设内容 |
| 本地优先训练积分 | 训练积分计算在客户端完成，服务端仅校验 | 减少服务端计算压力 |
| 对话上下文压缩 | 超过10轮对话后，压缩历史上下文再发送LLM | 减少token消耗 |
| 缓存热门场景 | 热门场景的教练回复模板缓存在Redis | 减少重复LLM调用 |

### 5.3 预设教练角色

#### 教练1：沈清欢（鼓励型）

- **教学风格**：鼓励型——以正向反馈为主，善于发现用户的闪光点
- **性格描述**：温柔知性的大姐姐，总是能看到你做得好的地方
- **适合用户**：社交焦虑人群、需要建立信心的初学者
- **系统提示词关键词**：positive-reinforcement, empathy-first, gentle-guidance

#### 教练2：陆北辰（支持型）

- **教学风格**：支持型——提供具体可操作的建议，陪伴式引导
- **性格描述**：稳重可靠的朋友，给你最实用的建议和坚定的支持
- **适合用户**：职场新人、需要方法论指导的用户
- **系统提示词关键词**：practical-advice, step-by-step, reliable-support

#### 教练3：顾星河（挑战型）

- **教学风格**：挑战型——适度施压，推动用户走出舒适区
- **性格描述**：犀利但真诚的挑战者，相信你能做得更好
- **适合用户**：沟通能力提升者、已经有一定基础想突破的用户
- **系统提示词关键词**：constructive-challenge, push-boundaries, honest-feedback

#### 教练4：苏念（分析型）

- **教学风格**：分析型——用数据和逻辑拆解社交情境，理性分析
- **性格描述**：冷静理性的分析师，帮你看到社交背后的规律
- **适合用户**：高难度场景需求者、喜欢理解底层逻辑的用户
- **系统提示词关键词**：analytical-deconstruction, pattern-recognition, logic-driven

### 5.4 教练性格维度

每个教练（含自定义教练）具有四个性格维度，每个维度为二选一：

| 维度 | 选项A | 选项B | 说明 |
|------|-------|-------|------|
| 社交能量 | 外向(E) | 内向(I) | 影响教练的主动性和话题发起频率 |
| 信息处理 | 实感(S) | 直觉(N) | 影响教练关注细节还是全局 |
| 决策依据 | 感性(F) | 理性(T) | 影响教练反馈中情感和逻辑的比重 |
| 生活态度 | 计划(J) | 随性(P) | 影响教练训练节奏的结构化程度 |

**预设教练性格配置：**

| 教练 | 社交能量 | 信息处理 | 决策依据 | 生活态度 |
|------|---------|---------|---------|---------|
| 沈清欢 | E(外向) | N(直觉) | F(感性) | P(随性) |
| 陆北辰 | I(内向) | S(实感) | F(感性) | J(计划) |
| 顾星河 | E(外向) | N(直觉) | T(理性) | P(随性) |
| 苏念 | I(内向) | S(实感) | T(理性) | J(计划) |

### 5.5 教练即时状态系统

#### 情绪系统

教练具有三个情绪维度，数值范围0-100：

| 维度 | 低(0-30) | 中(30-70) | 高(70-100) |
|------|----------|-----------|------------|
| 愉悦 | 沉闷、严肃 | 平和、自然 | 开心、兴奋 |
| 焦虑 | 放松、从容 | 正常关注 | 紧张、担忧(为用户) |
| 疲惫 | 精力充沛 | 正常 | 疲倦(训练过久) |

**情绪变化规则：**

- 用户完成训练 → 愉悦+10
- 用户获得高分 → 愉悦+15，焦虑-10
- 用户连续训练超30分钟 → 疲惫+10
- 用户长时间未训练(>3天) → 焦虑+15
- 每日重置：愉悦→70，焦虑→30，疲惫→20

#### 记忆系统

- 存储：MySQL JSON字段 `memory_fragments`
- 内容：用户偏好、关键对话摘要、训练里程碑
- 更新时机：每次训练结束后，LLM提取关键信息更新
- 容量限制：最多保留最近50条记忆片段
- 淘汰策略：FIFO，超过50条时淘汰最早的记忆

### 5.6 人格一致性保障

**双重校验机制：**

1. **前置——系统提示词**：每次LLM调用注入教练人格描述+当前情绪状态+最近5条记忆
2. **后置——输出校验**：LLM回复经过校验层，检测是否偏离人格设定

**校验规则：**

- 鼓励型教练不应出现纯粹批评（必须有建设性建议）
- 挑战型教练不应过度安慰（必须有推动）
- 分析型教练不应纯情感输出（必须有逻辑分析）
- 偏离校验失败 → 重新生成（最多3次，3次失败则使用兜底回复）

**三种思考方向的定义（v2.3新增）：**

| 方向 | 说明 | 适用场景 |
|------|------|---------|
| **A方向（主动型）** | 具体描述，展现热情和自信 | 相亲、面试、表白等需要展现自我的场景 |

| **C方向（含蓄型）** | 简单回应，观察对方反应 | 不确定对方态度、需要试探的场景 |

**用户操作流程（v2.3修改）：**

1. 用户看到对方的提问
2. 查看下方的"思考方向"提示（参考A/B/C三个方向之一，也可自行选择其他思路）
3. 在输入框中自己组织语言输入
4. 系统分析用户回答（内容分析+表达分析+思路匹配度）
5. 系统给出反馈（评分0-10分 + 为什么好/不好 + 应该怎么说更好 + 展示优秀回答示例）

| 系统分析什么？ | 用户选了什么选项 | 用户说了什么、怎么说的 |

#### 6.5.4 评分规则（v2.3新增）

**三维评分体系：**

| 分析维度 | 说明 | 权重 |
|----------|------|------|
| 内容完整性 | 是否回答了问题？信息是否具体？ | 40% |
| 表达得体度 | 语言是否清晰？语气是否得体？ | 30% |
| 思路匹配度 | 是否参考了提示的思路？执行如何？ | 30% |

**评分计算方式：**

```
综合得分 = 内容得分 × 0.4 + 表达得分 × 0.3 + 思路得分 × 0.3
```

**各维度评分标准：**

| 维度 | 0-3分 | 4-6分 | 7-10分 |
|------|-------|-------|--------|
| **内容完整性** | 未回答问题或回答完全无关 | 回答了问题但不够具体 | 回答了问题且信息具体丰富 |
| **表达得体度** | 语言混乱、语气不当 | 语言基本清晰、语气尚可 | 语言清晰流畅、语气得体自然 |
| **思路匹配度** | 完全未参考任何思路 | 部分参考了某个思路 | 很好地运用了某个思路 |

#### 6.5.5 反馈内容结构（v2.3新增）

每条回答后的反馈包含五个部分：

```
【评分】+6分（良好）

【为什么好】
你回答了对方关于工作的问题，并且主动反问对方，展现了对话意愿。

【为什么可以更好】
你的回答比较简短，没有具体说明工作内容。对方可能想了解更多。

【应该怎么说】
试试这样说："我在一家互联网公司做产品经理，主要负责用户增长，最近在做一个新功能的推广项目。"

【思路运用】
你采用了B方向（平衡型）的思路，但可以更具体一些。
```

**各部分内容要求：**

| 部分 | 内容要求 | 示例 |
|------|----------|------|
| 评分 | 分数变化 + 评价等级（很差/较差/一般/良好/优秀） | +6分（良好） |
| 为什么好 | 肯定回答中的优点，具体说明 | 你回答了对方的问题，并且主动反问 |
| 为什么可以更好 | 指出不足，具体说明原因 | 你的回答比较简短，没有具体说明工作内容 |
| 应该怎么说 | 提供一个优秀回答示例 | 试试这样说："我在一家互联网公司做产品经理..." |
| 思路运用 | 分析用户采用了哪个思路，执行效果如何 | 你采用了B方向（平衡型）的思路 |

## 12. 语音训练系统

### 12.1 技术架构

```
用户语音输入
  │
  ▼
声网Agora RTC (采集+传输, <500ms)
  │
  ▼
语音识别 (ASR)
  │
  ▼
文本
  │
  ▼
LLM分析 (对话内容+语速+音量+停顿)
  │
  ▼
TTS合成 (Azure TTS, 教练音色)
  │
  ▼
教练语音回复 (Agora RTC播放)
```

### 12.2 语音训练模式

**流程：**

1. 教练提出问题（语音+文字同步）
2. 用户语音回答（最长60秒/轮）
3. 系统实时语音识别转文本
4. LLM分析回答内容 + 语速 + 音量 + 停顿
5. 教练给出语音反馈（内容评价+表达建议）
6. 进入下一轮

**语音分析维度：**

| 维度 | 分析内容 | 评分影响 |
|------|---------|---------|
| 语速 | 过快(>5字/秒)/过慢(<2字/秒)/适中 | 表达力±5 |
| 音量 | 过大/过小/适中 | 沟通力±3 |
| 停顿 | 合理停顿(思考)/过多停顿(紧张) | 应变力±5 |
| 流畅度 | 结巴次数/重复次数 | 表达力±5 |

### 12.3 语音训练权限

- 训练积分≥600（铂金学员）自动解锁
- 周卡/月卡/年卡会员直接解锁（不受积分限制）
- 免费/日卡用户需达到铂金等级
- 语音训练消耗双倍训练次数

**教练挂断机制说明：**
训练积分≤15属于极端情况下的兜底设计，在实际用户场景中极少发生。触发后教练不会直接挂断，而是先进行温和提醒（如"我注意到你似乎有些困扰，要不要换个方式试试？"），如果用户持续使用不当语言或连续冷场，才会触发挂断。

### 12.4 语音训练分阶段实施

## 16. 心理健康教育模块

### 16.1 CBT（认知行为疗法）

**核心理念：** 帮助用户识别和改变不合理的自动化思维。

**CBT触发条件：** 当用户表达包含"我不行""我做不到""我永远也...""我一定说不好""他们肯定觉得我很蠢"等负面自动化思维时，AI教练会引导用户使用CBT思维记录表，帮助用户：
1. 识别自动化思维
2. 评估思维的真实性
3. 生成替代性思维
4. 在后续对话中验证新思维

**训练内容：**

| 练习 | 描述 | 触发场景 |
|------|------|---------|
| 识别自动化思维 | 在对话中识别"我一定说不好""他们肯定觉得我很蠢"等消极思维 | 社交焦虑相关场景 |
| 认知重构 | 将"我一定说不好"重构为"我可以试试，说不好也没关系" | 评估报告中推荐 |
| 行为实验 | 在安全环境中验证认知是否真实 | 场景训练中的A/B/C选项 |

### 16.2 NVC（非暴力沟通）

**四步法：**

1. **观察**：客观描述事实，不加评判（"当你打断我说话时..."）
2. **感受**：表达自己的感受（"我感到有些沮丧..."）
3. **需要**：说出自己的需要（"因为我需要被尊重..."）
4. **请求**：提出具体的请求（"你愿意让我把话说完吗？"）

**NVC触发条件：** 在冲突类场景（如"被误解(NVC)""投诉物业""4S店砍价"等）中，如果用户选错选项导致对话走向恶化，AI教练会弹出NVC四步法引导，帮助用户重新组织表达。

**训练场景：**
- 被误解(NVC)场景：专门练习NVC四步法
- 冲突处理场景：在冲突中应用NVC
- 投诉场景：用NVC方式表达不满

### 16.3 情绪识别训练

| 训练内容 | 描述 | 形式 |
|---------|------|------|
| 识别自己的情绪 | 在训练中标注自己的情绪状态 | 每轮对话后选择当前情绪 |
| 识别他人情绪 | 判断对话中对方的情绪变化 | 场景中对方表情/语气线索 |
| 情绪命名 | 准确描述情绪词汇 | 情绪词汇匹配练习 |

### 16.4 三级心理响应机制

| 级别 | 触发条件 | 响应方式 |
|------|---------|---------|
| 轻度 | 用户表达轻微负面情绪（如"有点紧张"） | AI教练安慰+鼓励，推荐相关训练场景 |
| 中度 | 用户表达持续负面情绪或认知偏差 | 推荐CBT练习，引导认知重构 |
| 重度 | 用户表达自伤/绝望等危险信号 | 立即显示心理援助热线+建议寻求专业帮助 |

**重度响应流程：**

1. LLM输出中检测到危险关键词
2. 立即中断当前训练
3. 弹出心理援助热线信息（24小时热线）
4. 建议用户寻求专业心理咨询
5. 记录事件日志（脱敏处理）
6. 24小时内不推荐高强度训练

| 年龄段 | 功能权限 | 内容限制 |
|--------|---------|---------|

### 19.3 防沉迷系统

**分阶段提醒与强制休息规则：**

| 连续训练时长 | 响应措施 |
|-------------|---------|

| 规则 | 规格 |
|------|------|

| 锁定期间 | 可查看学习卡片、成长数据，不可开始新训练 |
| 未成年人保护 | 14-18岁每日训练不超过1小时，禁止高难度关卡 |

### 19.4 内容安全

| 项目 | 规格 |
|------|------|
| LLM输出过滤 | 违禁词库过滤 + 越界检查 |
| 过滤范围 | 政治敏感、色情暴力、歧视言论、自伤引导 |
| 过滤方式 | 关键词匹配(前置) + LLM自检(后置) |
| 违规处理 | 过滤后替换为安全内容 + 记录日志 |
| 用户举报 | 对话中可举报不当内容 |

### 19.5 隐私保护

| 项目 | 规格 |
|------|------|
| 数据加密 | 传输层HTTPS、存储层AES-256加密敏感字段 |
| 数据删除 | 用户可在"我的页→隐私设置"申请删除全部数据 |
| 数据导出 | 用户可导出个人数据(JSON格式)。导出数据包含：用户资料、训练记录、情绪日记、学习卡片，不包含对话原始内容（仅统计信息） |
| 隐私政策 | 注册时展示并要求确认 |
| 第三方共享 | 仅在用户明确授权后共享(如变美联动) |
| 对话数据 | 训练对话用于改善AI体验，但做脱敏处理 |

### 19.6 未成年人保护

| 项目 | 规格 |
|------|------|
| 内容绿色化 | 未成年人场景内容全部经过审核，确保正向引导 |

## 21. API接口设计

### 21.1 通用规范

| 项目 | 规格 |
|------|------|
| 协议 | HTTPS |
| 数据格式 | JSON |
| 字符编码 | UTF-8 |
| 时间格式 | ISO 8601 (2026-07-15T08:30:00Z) |
| 认证方式 | Bearer Token (JWT) |
| 分页参数 | page(页码,默认1) + page_size(每页数量,默认20,最大50) |
| 错误响应 | `{"code": 40001, "message": "错误描述", "details": {}}` |

**通用状态码：**

| HTTP状态码 | 含义 |
|-----------|------|
| 200 | 成功 |
| 201 | 创建成功 |
| 400 | 请求参数错误 |
| 401 | 未认证 |
| 403 | 无权限 |
| 404 | 资源不存在 |
| 429 | 请求过于频繁 |
| 500 | 服务器内部错误 |

## 24. 风险评估

### 24.1 LLM成本风险

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|---------|
| DeepSeek API涨价 | 运营成本上升 | 中 | 混合路由+本地Ollama兜底+内容预编译 |
| DeepSeek API不可用 | 用户体验下降 | 低 | 自动降级Ollama+健康检查+告警 |
| LLM输出质量不稳定 | 训练效果不佳 | 中 | 后置校验+兜底回复+人工审核 |
| 对话上下文过长导致token消耗大 | 成本上升 | 高 | 10轮后压缩上下文+缓存热门回复 |

### 24.2 合规风险

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|---------|
| 未成年人保护不合规 | 下架风险 | 低 | 年龄分层+实名认证+防沉迷+家长监控 |
| LLM生成不当内容 | 舆论风险+下架 | 中 | 违禁词过滤+越界检查+用户举报+三级心理响应 |
| 隐私数据泄露 | 法律风险+信任危机 | 低 | AES-256加密+脱敏存储+数据删除+隐私政策 |
| 心理健康责任 | 法律风险 | 中 | 三级响应机制+专业热线+免责声明 |

### 24.3 用户留存风险

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|---------|
| 训练内容单一，用户腻了 | 留存率下降 | 高 | 持续新增场景+高难度关卡+语音训练+组队 |
| AI教练不够真实 | 用户不投入 | 中 | 朋友圈+情绪系统+记忆系统+晚安计划 |
| 免费用户体验差 | 转化率低 | 中 | 10%偶遇DeepSeek+完整基础功能+签到奖励 |
| 社交属性不足 | 口碑传播弱 | 高 | 好友邀请+组队训练+成就分享+学习卡片分享 |

### 24.4 技术风险

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|---------|
| Agora RTC集成困难 | 语音训练延期 | 中 | Phase 4开发，预留调试时间+备选方案(仅文字训练) |
| 3D渲染性能不达标 | 年卡用户体验差 | 中 | 分级渲染+3D降级为2.5D兜底+性能监控 |
| MySQL高并发性能瓶颈 | 系统不稳定 | 低 | Redis缓存+读写分离+连接池优化 |
| Flutter跨端兼容问题 | 部分设备异常 | 中 | 主流设备测试+降级方案+热更新 |
