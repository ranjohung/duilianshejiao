## 社交模拟场景系统

### 六大社交阶段

| 阶段 | 名称 | 核心能力 | 描述 |
|------|------|---------|------|
| 阶段一 | 破冰期 | 开启对话、第一印象 | 学习如何自然地开始一段对话 |
| 阶段二 | 接触期 | 维持对话、寻找共同话题 | 学习如何让对话持续下去 |
| 阶段三 | 熟悉期 | 深度交流、情感表达 | 学习如何表达真实的自己 |
| 阶段四 | 亲密期 | 冲突处理、深层共情 | 学习如何在矛盾中沟通 |
| 阶段五 | 职场期 | 职场沟通、专业表达 | 学习职场中的得体沟通 |
| 阶段六 | 高难期 | 压力沟通、谈判博弈 | 学习在高压场景中保持冷静 |

### 训练流程

```
开始训练
  │
  ▼
场景介绍（15秒）
  │  - 展示场景背景、对方角色、训练目标
  │
  ▼
对话轮次（核心环节，3-8轮）
  │  ┌─────────────────────────────────┐
  │  │  教练/AI角色说话                  │
  │  │         ↓                       │
  │  │  展示A/B/C三个选项               │
  │  │    A: 直接表达型                  │
  │  │    B: 委婉表达型                  │
  │  │    C: 回避/幽默型                 │
  │  │         ↓                       │
  │  │  用户选择 → 教练即时反馈           │
  │  │  （指出选择的优劣+改进建议）        │
  │  └─────────────────────────────────┘
  │
  ▼
评估报告
  │  - 总分(0-100)
  │  - 各维度得分
  │  - 星级评价(1-3星)
  │  - 优势亮点
  │  - 改进建议
  │  - 教练寄语
  │
  ▼
引导卡片
  │  - 本次训练的关键知识点
  │  - 可收藏、可分享
  │
  ▼
结束（返回场景页）
```

### 训练弹窗提示流程（v2.3新增）

```
用户点击"开始训练"
  │
  ▼
判断 hasStartedTrainingBefore
  │
  ├─ false（首次）→ 不弹窗，直接进入训练准备页
  │                   │
  │                   ▼
  │              userData.hasStartedTrainingBefore = true
  │
  └─ true（非首次）→ 显示好感度等级+权益弹窗
                       │
                       ▼
                   用户点击"开始训练"关闭弹窗
                       │
                       ▼
                   进入训练准备页
```

**好感度等级与权益对应表：**

| 等级 | 名称 | 图标 | 配色 | 已解锁权益 |
|---|---|---|---|---|
| Lv.0 | 初识 | 🌱 | 灰色 | 基础对话训练 |
| Lv.1 | 熟悉 | 💚 | 绿色 | 基础对话训练 + 示范回答 |
| Lv.2 | 信任 | 💙 | 蓝色 | 基础对话训练 + 示范回答 + 完整分析 |
| Lv.3 | 默契 | 🧡 | 橙色 | 基础对话训练 + 示范回答 + 完整分析 + 个性化教学 |
| Lv.4 | 知己 | 💖 | 粉色 | 全部教学功能 |

**弹窗组件：**

- 函数：`showTrainingReminderModal()`
- 配置函数：`getFavorabilityConfig(level)` 返回 `{name, icon, bgClass, bgLight, borderClass, textColor, features[], nextFeature}`
- 触发位置：`startTraining()` 函数中，检查 `userData.hasStartedTrainingBefore`
- 关闭方式：用户点击"开始训练"按钮，调用 `closeTrainingReminder()`

**选项设计规则：**

- A选项：直接/主动型——效果通常最积极，但需考虑场合
- B选项：委婉/间接型——社交安全但可能效果不足
- C选项：回避/幽默型——避免冲突但可能错过沟通机会
- 每轮选项设计必须体现不同的沟通策略，无绝对正确答案
- 选项效果根据场景上下文动态评估，不是固定得分

### 阶段解锁规则

| 阶段 | 解锁条件 |
|------|---------|
| 阶段一 | 默认解锁 |
| 阶段二 | 完成阶段一至少1个场景 + 训练积分 ≥ 50 |
| 阶段三 | 完成阶段二至少1个场景 + 训练积分 ≥ 100 |
| 阶段四 | 完成阶段三至少1个场景 + 训练积分 ≥ 150 |
| 阶段五 | 完成阶段四至少1个场景 + 训练积分 ≥ 200 |
| 阶段六 | 完成阶段五至少1个场景 + 训练积分 ≥ 300 |

**解锁校验逻辑：**

1. 查询用户训练积分
2. 查询用户在各阶段的完成记录
3. 同时满足积分+完成条件才解锁
4. 解锁后前端刷新场景列表
5. 解锁事件记入成就系统

### 解锁与通关条件

| 项目 | 条件 |
|------|------|
| 解锁条件 | 训练积分 ≥ 300 |
| 通关条件 | 单次训练获得三星评价 |
| 重试机制 | 无次数限制，但每次重试消耗1次训练次数 |
| 奖励 | 首次三星通关获得双倍训练积分 + 专属成就卡片 |

### 高难度关卡特殊设计

- **多分支结局**：根据关键选择节点，产生2-3种不同的对话走向
- **突发事件**：对话中随机插入意外情况（如对方突然发怒/提出额外要求）
- **时间压力**：部分关卡设置回复倒计时，模拟真实压力
- **NVC强制应用**：特定轮次要求使用NVC四步法，否则扣分

## 语音训练系统

### 技术架构

```
用户语音输入
  │
  ▼
声网Agora RTC (采集+传输, <500ms)
  │
  ▼
语音识别 (ASR)
  │
  ▼
文本
  │
  ▼
LLM分析 (对话内容+语速+音量+停顿)
  │
  ▼
TTS合成 (Azure TTS, 教练音色)
  │
  ▼
教练语音回复 (Agora RTC播放)
```

### 语音训练模式

**流程：**

1. 教练提出问题（语音+文字同步）
2. 用户语音回答（最长60秒/轮）
3. 系统实时语音识别转文本
4. LLM分析回答内容 + 语速 + 音量 + 停顿
5. 教练给出语音反馈（内容评价+表达建议）
6. 进入下一轮

**语音分析维度：**

| 维度 | 分析内容 | 评分影响 |
|------|---------|---------|
| 语速 | 过快(>5字/秒)/过慢(<2字/秒)/适中 | 表达力±5 |
| 音量 | 过大/过小/适中 | 沟通力±3 |
| 停顿 | 合理停顿(思考)/过多停顿(紧张) | 应变力±5 |
| 流畅度 | 结巴次数/重复次数 | 表达力±5 |

### 语音训练权限

- 训练积分≥600（铂金学员）自动解锁
- 周卡/月卡/年卡会员直接解锁（不受积分限制）
- 免费/日卡用户需达到铂金等级
- 语音训练消耗双倍训练次数

**教练挂断机制说明：**
训练积分≤15属于极端情况下的兜底设计，在实际用户场景中极少发生。触发后教练不会直接挂断，而是先进行温和提醒（如"我注意到你似乎有些困扰，要不要换个方式试试？"），如果用户持续使用不当语言或连续冷场，才会触发挂断。

### 语音训练分阶段实施

| 阶段 | 能力 | 说明 |
|------|------|------|
| MVP | AI教练语音播放(TTS) + 用户文字回复 | 教练用语音"说话"，用户仍用文字回复，降低技术复杂度 |
| V1.1 | 完整语音训练(用户语音输入+ASR识别+教练语音回复) | 集成Agora RTC+ASR，实现全双工语音对话 |
